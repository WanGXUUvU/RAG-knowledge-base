# TASK-085: 侧边栏工作区操作强化与顶奢 UI 视觉升级

## 1. 核心任务定义 (Task Refactoring)
**目标**：提升项目工作区与会话管理的易用性和美观度：
1. **侧边栏工作区快捷键 & 按钮**：在侧边栏每个工作区（文件夹）右侧，在 Hover 时浮现出 **【加号】(+)** 和 **【垃圾桶】(🗑️)**。加号用于直接在该工作区下新增一个会话；垃圾桶用于删除/清空该工作区下的所有会话，让工作区在侧边栏中视觉退场。
2. **移除顶部冗余绑定栏**：彻底砍掉聊天区顶部的 `workspace-binding-bar`，净化对话视觉，释放更多黄金纵向视界。
3. **顶奢视觉重设计**：对侧边栏的折叠文件夹头部样式进行升级，添加微光背景、动态 Hover 反馈和流畅过渡微动效，使其具有极高的高级质感。

---

## 2. 新任务拆解模板 (AGENTS.md Template)

### 用户动作：
1. 用户在侧边栏 Hover 到某个工作区目录（如 `test`）。
2. 用户点击该工作区右侧浮现的 **【+】** 按钮，或者点击 **【🗑️】** 垃圾桶按钮。
3. 用户在聊天界面与 AI 对话。

### 用户会看到：
1. **浮现操作按钮**：Hover 到工作区时，右侧平滑淡入两个高定制微光按钮：【+】和【🗑️】。
2. **新建会话**：点击【+】直接在该目录下瞬间拉起新会话并自动聚焦。
3. **删除确认与清空**：点击【🗑️】弹出极富设计感的二次确认框，确认后一键清空该工作区下的所有会话，该目录在会话树中完美淡出。
4. **聊天区极致净化**：对话顶部不再有大块的绿色/黄色长条绑定栏，取而代之的是纯粹、宽敞的无干扰对话黄金比例视域。

### 新数据从哪里产生 / 存在哪里：
1. 新会话创建：发起 `/run/session` 创建请求，归属于指定的工作区路径。
2. 批量删除：对该工作区下的所有 `session_id` 发起 `delete` 请求，在后端数据库和前端 Ref `sessions` 中清除。

### 前端调哪个接口 / need 改的层：
1. 前端循环触发 `/sessions/{id}` 接口进行会话批量删除。
2. 改造层：
   - `frontend/src/components/SessionSidebar.vue` -> 添加 `+` 与 `🗑️` 按钮、批量删除确认函数、Sleek CSS 样式。
   - `frontend/src/views/CodingView.vue` -> 移除聊天区顶部 `workspace-binding-bar` 元素，进一步优化布局高度。

## 3. 范围内与范围外 (In & Out of Scope)

### 范围内：
1. **文件夹操作功能**：
   - `+` 按钮一键在该文件夹下创建新会话（继承文件夹路径与名称）。
   - `🗑️` 按钮迭代调用 `emit('delete')` 逐个删除该工作区下的所有会话，并提供清空二次确认。
2. **UI 净化**：完全移除 `CodingView.vue` 顶部的绑定长条，将其所有的文件夹绑定交互（如绑定、切换项目目录）统一收纳进侧边栏的 Global Folder Selector 中。
3. **顶奢极客风格升级**：
   - 侧边栏工作区头部 Hover 时呈现拉丝微光渐变背景 (`background: var(--bg-hover)`)。
   - 动作按钮 Hover 时带有细微的半透明发光背景，删除按钮 Hover 呈现高级的警示红微光（`rgba(255, 69, 58, 0.15)`）。
   - 侧边栏整体更加匀称，过渡动效符合弹簧物理曲线。

### 范围外：
1. 物理删除磁盘上的文件夹。
2. 重构后端的 SQLite 工作区表。
