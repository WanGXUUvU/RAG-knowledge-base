正常完成

StreamRunSession._handle_completed() 调 RunPersistenceService.save_completed()，随后若有 staged VFS 则 commit。

审批暂停 / 恢复

暂停走 ToolRunObserver.handle_paused()；恢复完成走 save_resumed()；恢复后若还有下一个审批，则走 save_resumed_partial()。

用户取消

StreamRunSession.run() finally 分支调 save_cancelled()，再 RunVfsRegistry.discard(run_id)。

网络异常

当前实现里，模型网络异常也会落到同一个 finally 兜底，所以和取消共用 save_cancelled() 这条收口。

注：read_file 是 SAFE 工具，不会走审批。审批分支只是借它的同一条主运行骨架，展示高风险工具会在哪里暂停、恢复和落库。 另一个关键事实：当前“网络异常”和“取消”共享 save_cancelled() 收口，这也是后续要不要拆开 run_status 语义的重要设计点。
